### [Why is WebAssembly a second-class language on the web? - Mozilla Hacks - the Web developer blog](https://hacks.mozilla.org/2026/02/making-webassembly-a-first-class-language-on-the-web/)

WasmがまだWebにおいて第二級言語(second-class language。JSがfirst-class)とされている理由と、今後どう解決されていくか。
Wasmはブラウザで動くとはいっても大量のJSのglue codeを必要とする点で、DX的にもパフォーマンス的にもイマイチ。
このあたり解決するのがWebAssembly Component Modelですよという話。

将来的にはWeb APIをWasm実装側の言語で直接扱えるようになり、

```rust
// Rustの例
use std::web::console;

fn main() {
  console::log("hello, world");
}
```

読み込みもscriptタグで直接、という世界観になっていくらしい

```html
<script type="module" src="component.wasm"></script>
```
